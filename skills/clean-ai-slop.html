▸ 6단계 정리 패스

핵심 원리: 테스트로 동작 잠근 후, 6개 스멜을 순차적 패스로 정리. 실패 시 되돌립니다.
0
테스트로 동작 잠금

정리 전 테스트 통과 확인 — 회귀 기준점.

1
死 code 제거

사용하지 않는 함수, 변수, import, 리턴값.

2
과잉 주석 제거

명백한 코드를 설명하는 주석, LLM 필러.

3
불필요 추상화 제거

한 번만 쓰는 헬퍼, 과도한 인터페이스.

4
방어적 편집증 제거

불가능한 케이스, 과잉 검증, null 체크 폭포.

5
장황한 네이밍 정리

AnalyzeAndProcessTheData → processData

6
LLM 필러 아티팩트 제거

자체 설명 주석, 과도한 docstring.

동작을 테스트로 잠그기 전 어떤 정리도 시작하지 않습니다. AI가 touch한 파일만 대상으로 합니다.
