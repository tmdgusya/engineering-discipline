독립 6단계 정리

Clean AI Slop

AI 생성 코드의 교정적 정리. 동작을 보존하면서 LLM 특유의 패턴을 제거합니다. 회귀 테스트 우선, 단일 스멜 패스 규율.

언제 사용하나요?

대규모 AI 코드 생성 세션 이후. 또는 "정리해줘", "deslop", "AI 코드 정리"라고 말했을 때.

어떻게 동작하나요?

핵심 원리: 테스트로 동작을 먼저 잠근 다음, 6개 스멜 카테고리를 순차적 패스로 정리합니다. 각 패스 전후로 테스트를 실행하고 실패 시 되돌립니다.
0
테스트로 동작 잠금

정리 전 테스트를 실행해 통과 상태를 확인합니다. 회귀 기준점.

1
死 code 제거

사용하지 않는 함수, 변수, import, 리턴값.

2
과잉 주석 제거

명백한 코드를 설명하는 주석, "이 함수는 ~한다" 타입의 LLM 필러.

3
불필요 추상화 제거

한 번만 쓰는 헬퍼, 과도한 인터페이스, wrapper.

4
방어적 편집증 제거

불가능한 케이스 핸들링, 과잉 검증, null 체크 폭포.

5
장황한 네이밍 정리

AnalyzeAndProcessTheData → processData

6
LLM 필러 아티팩트 제거

"이 코드는 ~합니다" 타입의 자체 설명 주석, 과도한 docstring.

하드 게이트

동작을 테스트로 잠그기 전 어떤 정리도 시작하지 않습니다. AI가 touch한 파일만 대상으로 합니다.
