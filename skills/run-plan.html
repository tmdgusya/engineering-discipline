워크플로 Worker-Validator

Run Plan

Worker가 구현하고 Validator가 독립적으로 검증합니다. 실패 시 자동 재시도, 3회 실패 시 사용자에게 에스컬레이션.

언제 사용하나요?

Plan Crafting이 완료된 직후. 또는 이미 작성된 계획 문서를 실행해야 할 때.
  • "계획 실행해", "플랜 돌려줘"
  • plan-crafting 완료 후 자동 전환

어떻게 동작하나요?

핵심 원리: Validator는 Worker의 코드를 절대 보지 않습니다. 오직 계획의 목표와 수락 기준만으로 독립 검증합니다.
0
검증 인프라 발견 + 에이전트 탐색

프로젝트의 테스트 러너, 사용 가능한 에이전트/스킬을 파악합니다.

1
계획 사전 검토

실행 전 계획 자체의 이슈(파일 충돌, 순서 오류)를 검사합니다.

2
Worker-Validator 루프

각 작업이 3단계 서브에이전트를 거칩니다: 적합성 검사 → Worker 구현 → Validator 검토. 정보 격리가 핵심입니다 — Validator는 Worker의 결과를 보지 않습니다.

3
E2E 검증 게이트

모든 작업 완료 후 최종 통합 검증을 실행합니다.

?
재시도 → 에스컬레이션

Validator 실패 시 Worker에게 피드백과 함께 재시도. 3회 연속 실패 시 사용자에게 에스컬레이션.
