Run Plan

Worker가 구현하고 Validator가 독립 검증합니다. 실패 시 자동 재시도, 3회 실패 시 에스컬레이션.

▸ 언제 사용하나요?

Plan Crafting이 완료된 직후. 또는 이미 작성된 계획 문서를 실행해야 할 때.

▸ 어떻게 동작하나요?

핵심 원리: Validator는 Worker의 코드를 절대 보지 않습니다. 계획의 목표와 수락 기준만으로 독립 검증합니다.

검증 인프라 발견

테스트 러너, 사용 가능한 에이전트/스킬 파악.

계획 사전 검토

Worker-Validator 루프

적합성 검사 → Worker 구현 → Validator 검토. Validator는 Worker 결과를 보지 않습니다.

3
E2E 검증 게이트

모든 작업 완료 후 최종 통합 검증.
