Long Run

며칠에 걸친 실행을 오케스트레이션합니다. 디스크 체크포인트로 크래시에서 생존하고, 병렬 마일스톤을 worktree 격리로 실행합니다.

언제 사용하나요?

Milestone Planning이 완료된 직후. 또는 며칠에 걸친 대규모 리팩토링/기능 개발이 필요할 때.
  • "long run", "장기 실행 시작", "마일스톤 실행해줘"

어떻게 동작하나요?

핵심 원리: 모든 마일스톤이 독립적인 plan-crafting → run-plan → review-work 사이클을 거칩니다. 실패한 마일스톤은 후속 마일스톤을 차단합니다.
1
마일스톤 순차 실행

각 마일스톤: plan-crafting → run-plan → review-work → 체크포인트 저장.

2
재시도 에스컬레이션

재실행 → 재계획 → 중단 (3회 시도, 영구 카운터 유지).

3
병렬 마일스톤 (worktree 격리)

의존성이 없는 마일스톤은 별도 git worktree에서 동시에 실행됩니다.

4
크로스-마일스톤 통합 검사

각 마일스톤 완료 후 통합 검사를 실행합니다. 마일스톤 간 간섭을 탐지.

5
Final E2E 게이트

전체 완료 후 최종 End-to-End 검증. Simplify로 코드 품질 정리, Systematic Debugging으로 이슈 해결.
