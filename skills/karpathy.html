Karpathy Guidelines

코드 구현을 위한 예방적 가드레일 — 수술적 변경, 가정 검증, 범위 규율을 코딩 전과 코딩 중에 강제합니다.

언제 사용하나요?

기능 구현 전과 코딩 중에 작동합니다. LLM의 흔한 실수를 사전에 차단합니다.
  • 기능 구현을 시작하기 전
  • 기존 컨텍스트를 읽지 않고 코드를 생성하려 할 때
  • 코드 수정 시

어떻게 동작하나요?

핵심 원리: 예외 없는 5가지 하드 규칙. 각 규칙 위반 시 변경을 멈추고 규칙을 따르도록 수정합니다.
1
수술적 변경 — Minimum Edit Needed

목표를 충족하는 최소 변경만 합니다. 리팩토링은 별도 작업입니다.

2
기존 코드 읽기 전까지 수정 금지

수정할 파일의 현재 컨벤션을 먼저 파악합니다. 읽지 않고 추측하지 않습니다.

3
모든 가정 검증

API, 함수 시그니처, 동작 — 읽거나 grep으로 확인합니다. 추측으로 코딩하지 않습니다.

4
구체적 성공 기준 정의

코딩 전 "이렇게 되면 완료"를 명확히 합니다.

5
가상의 미래 문제 해결 금지

"나중에 필요할 것 같아서" 추가하는 추상화, 설정, 검증은 하지 않습니다.
