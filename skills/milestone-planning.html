Milestone Planning

5개 독립 리뷰어가 병렬로 리뷰하고, 결과를 합성해 최적화된 마일스톤 DAG를 만듭니다.

▸ 워크플로우 위치

Clarification(9-15) → Milestone Planning → Long Run
트리거
마일스톤 나눠줘 ultraplan
입력
Context Brief (복잡도 9-15)
출력
마일스톤 DAG + Conflict Resolution Log
5병렬 리뷰어
7최적 마일스톤 수
∅교차 오염 없음
1통합 검증

▸ 언제 사용하나요?

Clarification이 복잡(Complex, 9-15점) 판정을 내린 직후. 며칠에 걸친 작업이 필요할 때.

▸ 어떻게 동작하나요?

핵심 원리: 5개 리뷰어는 서로 정보 격리되어 교차 오염 없이 각자의 관점에서 비판합니다.
1
Problem Brief

문제를 프레임화 — 목표, 범위, 제약사항.

2
5개 병렬 리뷰어

Feasibility · Architecture · Risk · Dependency · User Value

3
합성 — 마일스톤 DAG

5개 결과를 Conflict Resolution Log와 종합. 통합 검증 마일스톤 추가.

4
독립 DAG 검증

순환 의존, 파일 충돌, 성공 기준 품질 검증.

5
사용자 승인 → Harness 잠금

승인된 마일스톤이 상태 디렉토리에 잠금. Long Run으로 진입.

마일스톤 수 7개 초과: 경고 / 10개 초과: 사용자 추가 승인 필요. 과도한 분할은 관리 오버헤드를 만듭니다.
