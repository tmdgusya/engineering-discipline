장기 실행 5개 병렬 리뷰어

Milestone Planning

5개의 독립적인 리뷰어 에이전트가 병렬로 리뷰하고, 결과를 합성해 최적화된 마일스톤 의존성 DAG를 만듭니다.

언제 사용하나요?

Clarification이 복잡(Complex) 판정(점수 9-15)을 내린 직후. 며칠에 걸친 작업이 필요할 때.
  • "마일스톤 계획해줘", "마일스톤으로 나눠줘", "ultraplan"
  • 작업이 2-3개 이상의 독립적 하위 태스크로 분해될 때

어떻게 동작하나요?

핵심 원리: 5개 리뷰어는 서로 정보 격리되어 있습니다. 교차 오염 없이 각자의 관점에서 계획을 비판합니다.
1
Problem Brief

문제를 프레임화합니다 — 목표, 범위, 제약사항.

2
5개 병렬 리뷰어 동시 생성

Feasibility — 실현 가능한가? / Architecture — 구조가 올바른가? / Risk — 위험 요소는? / Dependency — 의존성 순서가 맞는가? / User Value — 사용자에게 필요한가?

3
합성 — 마일스톤 DAG

5개 리뷰어 결과를 Conflict Resolution Log와 함께 종합. 자동 통합 검증 마일스톤이 마지막으로 추가됩니다.

4
독립 DAG 검증

메인 에이전트가 순환 의존, 파일 충돌, 성공 기준 품질을 독립적으로 검증합니다.

5
사용자 승인 → Harness 잠금

승인된 마일스톤이 harness 상태 디렉토리에 잠깁니다. Long Run으로 실행 진입.

마일스톤 수 가드

7개 초과: 경고 / 10개 초과: 사용자 추가 승인 필요. 과도한 분할은 관리 오버헤드를 만듭니다.
