HOW IT MOVES

  1. clarification질문과 코드 탐색으로 요구사항을 고정
  2. plan-crafting / milestone-planning단순 작업과 장기 작업을 자동 분기
  3. run-plan / long-run실행과 재시도, 오케스트레이션
  4. review-work정보 격리된 상태로 PASS/FAIL 검증

01 — WORKFLOW

무질서를 줄이는 실행 경로

이 플러그인은 단순히 스킬 목록을 제공하지 않습니다. 요청을 해석하고, 난이도에 따라 경로를 바꾸고, 마지막에는 독립 검증으로 종료하는 실행 파이프라인을 제공합니다.

핵심 원리: 모든 전환점에 정보 격리와 검증 가능성을 심습니다. 구현자는 만들고, 검증자는 결과를 확인합니다.

STEP 1

Clarification

반복 질문 + 코드베이스 탐색으로 “무엇을 만들지”를 먼저 고정합니다.

  • 요구사항 명확화
  • 범위 축소
  • 복잡도 평가

STEP 2A

Plan Crafting

단순 작업은 즉시 실행 가능한 구현 계획으로 바꿉니다.

  • 실제 코드 포함 계획
  • 플레이스홀더 금지
  • 작업 순서 명확화

STEP 2B

Milestone Planning

큰 작업은 5개 병렬 리뷰 관점으로 분해해서 마일스톤 DAG로 정리합니다.

  • 실현 가능성
  • 리스크와 의존성
  • 사용자 가치 기준 분해

STEP 3

Run Plan / Long Run

짧은 작업은 worker-validator 루프로, 긴 작업은 체크포인트 기반 오케스트레이션으로 실행합니다.

  • 재시도 루프
  • 체크포인트 복구
  • 병렬 worktree 실행

STEP 4

Review Work

실행 로그가 아니라 계획과 결과만으로 검증해 PASS/FAIL 판정을 내립니다.

  • 정보 격리
  • 계획 대비 구현 검토
  • 자동 에스컬레이션 기준

02 — SKILL MAP

워크플로우에 연결된 스킬들

각 스킬은 단독으로도 쓸 수 있지만, 가장 큰 가치는 서로 연결될 때 나옵니다. 출발점, 계획, 실행, 검토, 장기 실행, 품질 보강까지 하나의 체계로 설계되어 있습니다.

WORKFLOW SKILLS

체인의 중심

Clarification 모호한 요청을 명확한 범위로 축소 Plan Crafting 바로 실행 가능한 다단계 계획 생성 Run Plan worker-validator 구현 루프 Review Work 정보 격리된 독립 검증

LONG RUNNING

복잡한 작업용

Milestone Planning 5개 병렬 리뷰어로 마일스톤 합성 Long Run 다일 작업 오케스트레이션과 체크포인트

INDEPENDENT SKILLS

언제든 호출

Karpathy 수술적 변경과 범위 규율 가드레일 Clean AI Slop AI 생성 코드 정리 Simplify 재사용성·품질·효율성 리뷰 Systematic Debugging 재현 우선, 근본 원인 우선 버그 수정 Rob Pike's 5 Rules 측정 기반 성능 최적화
